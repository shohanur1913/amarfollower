## Part 1: Public REST API for Users (Kotlin App Compatible)

### Architecture
- **Base URL:** `https://amarfollower.com/api/v1`
- **Auth:** `Authorization: Bearer <api_key>` header (ApiKey model use kore, যেটা already `api_keys` table e exist kore)
- **Rate Limiting:** Per-IP + Per-API-Key basis (existing `rate-limit.ts` + `api-helpers.ts` use kore)
- **Response Format:** Standard JSON `{ success: true/false, data: ..., error: ... }`

### API Authentication Middleware
`src/lib/api-auth.ts` — New file:
- `Authorization: Bearer <key>` header read kore
- `api_keys` table theke key lookup
- Check: `isActive`, `expiresAt` (not expired), user `status = active`
- `lastUsed` update kore
- IP whitelist check (if `ipWhitelist` field set)
- Returns authenticated user object or 401 error

### API Endpoints

#### 1. Auth Endpoints (Public, no API key needed)
| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/api/v1/auth/register` | Register new user |
| POST | `/api/v1/auth/login` | Login with email/password → returns JWT token |
| POST | `/api/v1/auth/refresh` | Refresh JWT token |

#### 2. Service Catalog (Public, no API key needed)
| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/v1/services` | All active services (with pagination, filter by platform/category) |
| GET | `/api/v1/services/:id` | Single service details |
| GET | `/api/v1/platforms` | All platforms with categories |
| GET | `/api/v1/gateways` | Active payment gateways |

#### 3. User Endpoints (API key required)
| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/v1/user/profile` | User profile info |
| PUT | `/api/v1/user/profile` | Update profile |
| GET | `/api/v1/user/balance` | Current balance |
| GET | `/api/v1/user/stats` | Dashboard statistics |

#### 4. Order Endpoints (API key required)
| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/api/v1/orders` | Create new order |
| GET | `/api/v1/orders` | List orders (pagination, filter by status) |
| GET | `/api/v1/orders/:id` | Single order details |
| POST | `/api/v1/orders/mass` | Mass/bulk order |

#### 5. Payment Endpoints (API key required)
| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/api/v1/payments` | Initiate payment |
| GET | `/api/v1/payments` | Payment history |

#### 6. Refill Endpoints (API key required)
| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/v1/refills` | List refills |
| POST | `/api/v1/refills` | Request refill |

#### 7. Ticket Endpoints (API key required)
| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/v1/tickets` | List tickets |
| POST | `/api/v1/tickets` | Create ticket |
| GET | `/api/v1/tickets/:id` | Ticket with messages |
| POST | `/api/v1/tickets/:id/reply` | Reply to ticket |

#### 8. Scheduled Order Endpoints (API key required)
| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/v1/scheduled-orders` | List scheduled orders |
| POST | `/api/v1/scheduled-orders` | Create scheduled order |
| DELETE | `/api/v1/scheduled-orders/:id` | Cancel scheduled order |
